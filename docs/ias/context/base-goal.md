## Goal

Daily Habit Tracker MVP (Refined) # Overview This proposal outlines the development of a modern, React-based Habit Tracker. The application focuses on building consistency through visual feedback, streak mechanics, and data-driven insights, utilizing a local-first architecture for privacy and speed. ## Objectives - Provide a seamless CRUD interface for habit management using Tailwind CSS and Shadcn/UI. - Implement robust streak logic based on the user's local browser timezone. - Visualize progress through interactive Recharts dashboards and heatmaps. - Ensure data persistence via LocalStorage with a focus on performance. ## Requirements - **Functional**: Create/Edit/Delete habits, toggle daily completion, calculate current/longest streaks, generate weekly/monthly reports. - **Non-Functional**: Responsive design (mobile-first), <2s load time, accessible UI components (Radix/Shadcn). ## Scope (In / Out) ### In-Scope - **Habit Management**: Custom names, descriptions, and categories. - **Streak Engine**: Logic for consecutive days and history tracking. - **Analytics Dashboard**: Recharts integration for completion rates and trend lines. - **Persistence**: LocalStorage with JSON serialization. - **Styling**: Tailwind CSS + Shadcn/UI for a clean, modern aesthetic. ### Out-of-Scope - Cloud synchronization (Supabase/Firebase) - deferred to V2. - Social sharing or community leaderboards. - Native mobile app wrappers (Capacitor/React Native). ## Plan (Phased) 1. **Phase 1: Foundation**: Initialize React + Tailwind + Shadcn. Setup LocalStorage hooks. 2. **Phase 2: Habit Core**: Build habit creation forms and the daily checklist view. 3. **Phase 3: Streak Logic**: Implement timezone-aware streak calculations and historical data storage. 4. **Phase 4: Visualization**: Build the Analytics tab using Recharts (Pie charts, Line graphs, Heatmaps). 5. **Phase 5: Reporting**: Logic for generating and displaying weekly/monthly performance summaries. ## Acceptance Criteria - Users can create a habit and see it on their daily dashboard. - Marking a habit as 'done' increments the streak and updates the Recharts visualization in real-time. - Data remains intact after a browser refresh. - The UI passes basic accessibility checks (WCAG AA) and is responsive down to 320px width. ## Risks and Mitigations - **Data Volatility**: LocalStorage can be cleared. *Mitigation*: Implement a 'Download Backup' feature (JSON). - **Timezone Edge Cases**: Users crossing date lines. *Mitigation*: Use `date-fns-tz` to anchor calculations to the browser's reported local time. - **Performance**: Large history objects. *Mitigation*: Memoize chart data and implement basic data pruning for records older than 1 year. ## Validation and Rollout - **Validation**: Unit tests for streak logic; manual testing across Chrome, Safari, and Firefox. - **Rollout**: Deploy to Vercel as a static site; initial 'Alpha' release to gather feedback on the UI/UX.
