# Persona passes

Run these passes before calling something “good enough”.

## User pass

- [ ] The critical path is obvious and low-friction.
- [ ] Copy/labels are clear; the user never feels lost.
- [ ] The experience “feels good” for the context (speed, polish, trust).

## PM / value pass

- [ ] This matches the base goal and success criteria.
- [ ] Scope is appropriate; no unnecessary complexity.
- [ ] Tradeoffs are intentional and documented.

## Implementation pass

- [ ] Core logic is tested where meaningful.
- [ ] Changes are maintainable and follow repo conventions.
- [ ] Risks are identified and mitigated or explicitly logged as gaps.

## Review pass

- [ ] No P0/P1 issues remain unaddressed.
- [ ] Decisions/gaps are tracked; merge blockers are resolved.
