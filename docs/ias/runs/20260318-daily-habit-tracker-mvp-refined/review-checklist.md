# Review checklist

## Summary

What changed and why.

## Correctness

- [ ] Behavior matches acceptance criteria.
- [ ] No broken build.

## Risk

- [ ] Any “one-way door” decisions are called out.
- [ ] Production/deployed-version implications are addressed (if applicable).

## Tests

- [ ] Core logic has meaningful automated tests.
- [ ] Tests are not “toy tests” (they would catch a real regression).

## UX clarity (if user-facing)

- [ ] The user can understand what to do without explanation.
- [ ] The critical path is short; no unnecessary steps.

## Maintainability

- [ ] Code is readable and consistent.
- [ ] Complexity is justified; alternatives considered if needed.

## Decisions and gaps

- [ ] New decisions are recorded (if any).
- [ ] Gaps are recorded (if any).
- [ ] No unresolved merge blockers.
